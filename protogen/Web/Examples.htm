Making Examples

One of the best ways to document a protocol is to show examples of the protocol in use. But protocol examples that are wrong or outdated can create more confusion than understanding.

Using the tools provided in ProtoGen and Goedel allows us to create protocol examples that are always synchronized to the latest version of the code. To do this we need:

  • A client to create requests.
  • A service to create responses.
  • A tool to merge the example messages into the documentation.

We have already seen how to create a client and server for the Enquirer service. We could use the same code to generate the protocol messages but this would require the program that creates the message examples to create separate threads for the client and server, pass messages between them over TCP/IP and somehow catch the messages in the wireline format.

The DirectBinding class provides a simpler way to generate sample messages by connecting the client transaction API to the server callbacks directly. This allows us to avoid the unnecessay complication of writting code to handle concurrency to create a few dozen sample messages.
